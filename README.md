# TaskSync - Full-Stack Task Management System

A production-ready, secure, and responsive full-stack Task Management application built as part of a Backend Developer Internship assignment. It integrates a secure RESTful API with a professional React frontend, featuring role-based authorization and real-time state synchronization.

---

## Project Overview

This monorepo project comprises two primary components:
1. **Backend Service**: A robust Node.js and Express RESTful API utilizing MongoDB for structured data storage, featuring password hashing, JWT authorization, model validation, and centralized exception handling.
2. **Frontend client**: A professional React + Vite single-page application styled using pure Vanilla CSS, utilizing React Router DOM for route security and Axios for proxied API requests.

---

## Features

### Backend Architecture
* **User Registration & Log In**: Secure registration and session initialization.
* **JWT Authentication**: Secure stateless token issuance and verification middleware.
* **Password Cryptography**: Strong hashing using `bcryptjs` before storage.
* **Role-Based Access Control (RBAC)**: Distinct permissions for `user` and `admin` roles.
* **Task CRUD APIs**: Clean endpoints for managing task lifecycles with authorization checks.
* **Request Validation**: Schema constraints on request payloads via `express-validator`.
* **Centralized Error Handling**: Standardized error responses to prevent sensitive stack leaks.
* **Interactive API Docs**: Real-time documentation and testing sandbox generated by Swagger UI.

### Frontend Client
* **React + Vite App**: Ultra-fast hot module replacement (HMR) and lightweight builds.
* **State Authentication**: Persistent login status using JWT stored in local storage.
* **Protected Routes**: Custom guards protecting dashboards from unauthorized lookups.
* **Auto-Logout Integration**: Global Axios interceptor automatically cleaning up local storage and logging out on 401 Unauthorized codes.
* **User Dashboard**: Core view enabling task filters, creation, status toggles, edits, and confirmations before deletion.
* **Admin Dashboard Panel**: Protected panel displaying system stats cards (Total, Pending, Completed), live title search, and administrative task management tables.
* **Responsive Layouts**: Consistent grid adjustments tailored for Mobile, Tablet, and Desktop screens.

## API Endpoints

### Authentication Routes
* `POST /api/v1/auth/register` - Registers a new user account.
* `POST /api/v1/auth/login` - Authenticates credentials and issues a JWT token.

### Tasks Management Routes (Authenticated)
* `POST /api/v1/tasks` - Creates a new task (Defaults status to `pending`).
* `GET /api/v1/tasks` - Fetches all tasks associated with user (Admins get all tasks globally).
* `GET /api/v1/tasks/:id` - Retrieves detailed statistics for a specific task.
* `PUT /api/v1/tasks/:id` - Updates task title, description, or status (Toggled between `pending` / `completed`).
* `DELETE /api/v1/tasks/:id` - Deletes a specific task.

## Installation

### Backend Setup
1. Navigate to the root directory:
   ```bash
   cd task-manager
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Set up the environment variables (see below).

### Frontend Setup
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install frontend dependencies:
   ```bash
   npm install
   ```
3. Set up the environment variables (see below).

JWT_EXPIRES_IN=7d
```

```

## Running Locally

### Start Backend Server
In the root directory, execute:
```bash
npm run dev
```
The server will initialize on `http://localhost:5000` and connect to the local MongoDB database.

### Start Frontend Server
In a separate terminal, navigate to the `frontend/` folder and run:
```bash
npm run dev
```
The Vite development server will spin up on `http://localhost:5173`. Any API calls to `/api/v1` will be proxied automatically to the backend port.

## Scalability Considerations

To progress this system to support millions of requests, the following architectural steps should be implemented:

1. **Redis Caching**: Cache repetitive query lookups (e.g., retrieving lists of completed tasks or dashboard statistics) in-memory using Redis. Establish standard Cache-Aside strategies to invalidate items when tasks are edited or deleted.
2. **Rate Limiting**: Integrate `express-rate-limit` on the authentication controller and task CRUD handlers to shield endpoints against Brute-Force intrusions and Denial of Service (DoS) attacks.
3. **Docker Containerization**: Dockerize the application by adding custom Dockerfiles for both frontend and backend modules, orchestrating them via `docker-compose` to enforce predictable sandbox runs across various staging levels.
4. **CI/CD Pipelines**: Incorporate automated pipelines (e.g., GitHub Actions) to run syntax linters, execute unit tests, and deploy builds automatically upon merged pull requests.
5. **Load Balancing**: Place an Nginx reverse proxy or AWS Application Load Balancer (ALB) ahead of the backend servers. Scale backend container nodes horizontally and distribute user request loads round-robin.
6. **Microservices Migration**: Decouple the monolithic Express app into smaller, domain-oriented services (e.g., Auth service, Task processor service, Notifications service) coordinating via message brokers like RabbitMQ or Apache Kafka.
